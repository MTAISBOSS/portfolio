Building an ECS Game Engine

A deep dive into Entity Component System architecture, data-oriented design, and building a modular game engine from scratch.

C++ ECS OpenGL

Why ECS?

Traditional object-oriented game engines use inheritance hierarchies—a Player inherits from Character, which inherits from Entity. This quickly becomes unwieldy. ECS solves this through composition over inheritance.

ECS isn't just an architecture pattern—it's a way of thinking about game objects as bags of data rather than class hierarchies.

The Core Architecture

Entities

In my implementation, an entity is simply a 32-bit integer ID. This keeps entities lightweight and allows millions without memory overhead.

using Entity = uint32_t;
const Entity MAX_ENTITIES = 1000000;

class EntityManager {
    std::queue<Entity> availableEntities;
    uint32_t livingEntityCount = 0;
    
public:
    Entity createEntity();
    void destroyEntity(Entity entity);
};

Components

Components are plain structs containing only data—no methods. They're stored in tightly-packed arrays for cache efficiency:

struct Transform {
    Vec3 position;
    Quat rotation;
    Vec3 scale;
};

struct Velocity {
    Vec3 linear;
    Vec3 angular;
};

Systems and Queries

Systems iterate over entities with specific component combinations:

class PhysicsSystem : public System {
public:
    void update(float dt) override {
        for (Entity entity : entities) {
            auto& transform = getComponent<Transform>(entity);
            auto& velocity = getComponent<Velocity>(entity);
            transform.position += velocity.linear * dt;
        }
    }
};

Data-Oriented Design

ECS naturally leads to data-oriented design. Benefits include:

  • Cache efficiency: Sequential memory access patterns
  • SIMD opportunities: Process 4-8 components per instruction
  • Easy parallelization: Systems can run concurrently

Performance Results

Compared to my previous inheritance-based engine:

  • 10x more entities at 60 FPS (10,000 → 100,000)
  • 50% reduction in frame time variance
  • Significantly easier to add new features
